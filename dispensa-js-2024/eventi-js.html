Introduzione agli Eventi in JavaScript

Gli eventi in JavaScript sono azioni o occorrenze che accadono nel browser, come un clic del mouse, una pressione di tasti, il caricamento di una pagina, il ridimensionamento di una finestra, e così via. JavaScript può essere usato per ascoltare questi eventi e rispondere in modo appropriato, creando interfacce utente interattive e dinamiche.

Tipi di Eventi Comuni

JavaScript supporta una vasta gamma di eventi. Ecco alcuni dei più comuni:

  • Eventi del Mouse: click, dblclick, mouseover, mouseout, mousedown, mouseup, mousemove
  • Eventi della Tastiera: keydown, keyup, keypress
  • Eventi di Form: submit, change, focus, blur
  • Eventi del Documento/Window: load, resize, scroll, unload

Aggiungere Event Listener

Per ascoltare un evento, si utilizza il metodo addEventListener, che permette di assegnare un gestore di eventi a un elemento del DOM. Il gestore di eventi è una funzione che viene eseguita quando l'evento specificato si verifica.

                
                    const bottone = document.getElementById('mioBottone');
                    bottone.addEventListener('click', function() {
                        alert('Bottone cliccato!');
                    });

Rimuovere Event Listener

Puoi rimuovere un event listener utilizzando il metodo removeEventListener. È importante che la funzione del gestore di eventi sia dichiarata separatamente, in modo da poterla passare a removeEventListener.

                
                    const bottone = document.getElementById('mioBottone');
                    
                    function mostraMessaggio() {
                        alert('Bottone cliccato!');
                    }
                    
                    bottone.addEventListener('click', mostraMessaggio);
                    bottone.removeEventListener('click', mostraMessaggio);

Event Object

Quando un evento si verifica, il browser crea un oggetto evento che contiene informazioni sull'evento. Questo oggetto è passato come argomento alla funzione del gestore dell'evento. L'oggetto evento contiene proprietà e metodi che possono essere utilizzati per ottenere ulteriori informazioni sull'evento e per controllare il comportamento dell'evento.

                
                    const bottone = document.getElementById('mioBottone');
                    
                    bottone.addEventListener('click', function(event) {
                        console.log('Tipo di evento:', event.type);
                        console.log('Elemento target:', event.target);
                    });

Event Delegation

Event delegation è una tecnica per gestire eventi in modo efficiente. Consiste nell'assegnare un gestore di eventi a un elemento padre e sfruttare il bubbling degli eventi per catturare gli eventi degli elementi figli. Questo è particolarmente utile quando si gestiscono eventi per elementi dinamici o numerosi.

                
                    const lista = document.getElementById('lista');
                    
                    lista.addEventListener('click', function(event) {
                        if (event.target.tagName === 'LI') {
                            alert('Elemento cliccato: ' + event.target.textContent);
                        }
                    });

Prevent Default e Stop Propagation

In alcuni casi, potresti voler impedire il comportamento predefinito di un evento o interrompere la propagazione dell'evento. Questo può essere fatto utilizzando i metodi preventDefault e stopPropagation dell'oggetto evento.

Prevent Default

Il metodo preventDefault impedisce il comportamento predefinito dell'evento. Ad esempio, puoi impedire che un link venga seguito o che un modulo venga inviato:

                
                    const link = document.getElementById('mioLink');
                    
                    link.addEventListener('click', function(event) {
                        event.preventDefault();
                        alert('Link cliccato, ma non seguito!');
                    });

Stop Propagation

Il metodo stopPropagation impedisce la propagazione dell'evento agli elementi parenti. Questo è utile quando si vuole gestire un evento in modo specifico senza che interferisca con altri gestori di eventi sugli elementi parenti:

                
                    const contenitore = document.getElementById('contenitore');
                    const bottone = document.getElementById('mioBottone');
                    
                    contenitore.addEventListener('click', function() {
                        alert('Contenitore cliccato!');
                    });
                    
                    bottone.addEventListener('click', function(event) {
                        event.stopPropagation();
                        alert('Bottone cliccato!');
                    });

Lista di eventi utilizzati

Eventi dell'interfaccia utente

  • load: utilizzato principalmente quando un elemento di una pagina web ha terminato il caricamento dei contenuti.
  • unload: utilizzato principalmente quando l'elemento di una pagina web prima che la finestra venga chiusa, l'elemento continua la sua operazione prima che si chiuda del tutto.
  • error: utilizzato principalmente dal browser quando lo script incontra un errore.
  • resize: utilzzato quando un elemento si comporta in modo diverso quando la finestra del browser viene ridimensionata.
  • scroll: utilizzato quando la pagina web viene scrollata verso il basso o alto.

Eventi di un modulo

  • focus: utilizzato per mettere a fuoco un elemento quando si utilizza.
  • blur: utilizzato per rilasciare l'elemento messo a fuoco quando non si utilizza.
  • value: utilizzato per prendere il valore di un elemento generale in campo di testo ad esempio.
  • submit: utilizzato principalmente per controllare i valori messi dall utente prima di essere inviati al server.
  • change: utilizzato principalmente per cambiare gli elementi selezionati da un nodo per varie condizioni.
  • input: utilizzato principalmente per i campi input e textarea a seconda delle varie situzioni.

Eventi di tastiera e mouse

  • click: utilizzato per cliccare una volta sola su quel elemento.
  • dblclick: utilizzato per cliccare due vole su quel elemento.
  • mousedown: utilizzato principalmente quando l'utente clicca su qualsiasi tasto del mouse
  • mouseup: utilizzato quando l'utente rilascia il tasto del mouse.
  • mouseover e mouseout: queste proprietà sono riassumibili con la pseudoclasse :hover del css.
  • oggetto evento per il mouse: tramite l'oggetto evento possiamo sapere dove si trova il mouse dallo schermo del pc alla finestra del browser: 'Screen', 'Page' e 'Client' con le cordinate X e Y.
  • keydown: utilizzato quando l'utente preme un tasto della tastiera.
  • keypress: utilizzato quando l'utente premendo un tasto della tastiera provaca la visualizzazione del carattere in quel momento.
  • keyup: utilizzato quando l'utente rilascia il tasto della tastiera.

Eventi di mutazione e di HTML5

  • DOMNodeInserted: utilizzato principalmente per inserire un nodo al interno del albero DOM.
  • DOMNodeRemoved: utilizzato principalmente per eliminare un nodo dal DOM
  • DOMSubtreeModified: utilizzato principalmente per camabiare la struttura del DOM.
  • DOMNodeInsertedIntoDocument: utilizzato principalmente per inserire un nodo all'interno di un altro nodo come discendente del documento.
  • DOMNodeRemovedIntoDocument: utilizzato principalmente per rimuovere un nodo all'interno di un altro nodo come discendente del documento.
  • DOMContentLoaded: utilizzato principalmente per anticipare il caricamento di un elemento del DOM che sia uno script o un risorsa.
  • hashchange: utilizzato principalmente per verificare se url della pagina cambia.
  • beforeunload: utilizzato principalmente tramite l'oggetto Window prima che la pagina venga scaricata attraverso delle condizioni.
